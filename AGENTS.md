# AGENTS.md — instructions for coding agents

Read this before changing anything in this repository.
This file follows the [AGENTS.md](https://agents.md) convention and is read by most coding
agents. Claude Code reads it through the import in `CLAUDE.md` (local, untracked).

## Project

- **What it is**: a client-side fee quote & split calculator for bridging EURC between Base
  and Stellar through Rozo's intents API — exact live quotes, a best-split recommendation, and
  live intent tracking. Zero backend, zero API key.
- **Stack**: static page, vanilla JS (`web/assets/`); Node.js dev scripts (built-ins only, no
  runtime dependencies); esbuild to regenerate the vendored Stellar Wallets Kit bundle.

## Commands

```bash
cd web && python3 serve.py   # run locally — serves web/assets/ and opens the page
npm test                     # fee-model self-check (node scripts/test-model.mjs)
npm run check:i18n           # fr/en dictionary parity (node scripts/check-i18n.mjs)
npm run build:walletkit      # regenerate the vendored web/assets/walletkit.js bundle
```

## Structure

- `web/` — the static app: `rozo-bridge.html`, `assets/` (JS), `serve.py` (local server).
- `CLI/` — a shell companion (`rozo-quote.sh`) that queries the same quote endpoint.
- `scripts/` — Node dev tooling: `build-walletkit.mjs`, `test-model.mjs`, `check-i18n.mjs`.
- `data/` — recorded intent log (`intent-log.jsonl`).

## Branching

GitHub Flow: `main` is always deployable.

- Branch off `main`: `feat/…` or `fix/…`.
- Open the pull request against `main`. The CI must be green before it is merged.
- **There is no staging branch, on purpose**: there is no host to validate against here.

Merging into `main` deploys the site: GitHub Pages serves it directly.

## Conventions

- **Everything committed is written in English** — code, code comments, docs, `README.md`,
  `.env.example`. The only exception is `README.md`, which is bilingual (English, then French).
- **Never use the second person** (`you`/`your`) in committed content or in the app's UI.
  Write "the user", or use impersonal phrasing.
- **i18n**: translations live in a separate dictionary — never inline ternaries in the markup.
- **`CHANGELOG.md`**: update the `Unreleased` section for any user-facing change. The GitHub
  Release carries the auto-generated list of merged pull requests; the changelog says what
  actually changed for a user.
- **Architecture decisions**: a non-trivial decision (stack, data source, boundary) gets a
  short record in `docs/adr/`. The point is to preserve the *why*, which the code cannot express.

## Merge discipline — the server enforces it

This repository is **public**, so rulesets protect `main`: a pull request is required, the
required checks must be green, and the server refuses a direct push or the merge of a red pull
request. Still, verify before merging — **a check MISSING from the list is not a green**, it
has simply not reported yet:

```bash
sha=$(gh pr view <n> --json headRefOid --jq .headRefOid)
gh run list --commit "$sha" --json workflowName,status,conclusion
```

> **Match on `workflowName`, never on `name`.** CodeQL runs through GitHub's *default setup*, so
> its run has no workflow file and its `name` field reads `Push on main` — only `workflowName`
> says `CodeQL`. `gh pr checks` cannot be used here: it needs the `Checks` permission, which does
> not exist in the fine-grained token UI (github/community#129512).

## Checks that run

- **pre-commit hook** — `gitleaks` on staged files (a commit carrying a secret is rejected), then a
  throttled, CONSULTATIVE replay of `./check.sh` (≤ once / 24 h) — it surfaces drift, never blocks.
  Tune the window with `CHECK_MAX_AGE_HOURS`.
- **pre-push hook** — refuses a direct push to `main`. A fresh clone re-arms both hooks once:
  `git config core.hooksPath .githooks`.
- **`./check.sh`** — replays the CI's security checks locally at the pinned versions, so
  `local == github`. It reads the pinned versions from `ci.yml` (single source) and caches the
  binaries under `.ci-tools/` (gitignored). The CI stays the authority: it alone verifies the
  SHA256 of the Linux assets.
- **CI** (on every pull request, required before merge) — `gitleaks` over the *full* history,
  `actionlint` + `zizmor` on the workflows, `semgrep` static analysis, `osv-scanner` on every
  manifest it discovers (`-r .`; CI-only tooling is out of scope via `.gitignore`), then the
  project's own checks (JS syntax, i18n parity, fee-model self-check).
- **CodeQL** — security analysis via default setup (active because the repository is public);
  a finding blocks the merge.

## Do not touch

- `web/assets/walletkit.js` — vendored Stellar Wallets Kit bundle, regenerated by
  `scripts/build-walletkit.mjs`. Edit the build script, never the bundle by hand.
- Never commit a secret. `.env` and `.envrc` are untracked, and must stay that way.
